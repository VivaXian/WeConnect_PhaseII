# WeConnect Phase II｜页面规格说明

> 本文件描述每个页面的内容结构、交互原则和角色差异。不涉及视觉设计细节，不规定组件 API。  
> 最后更新：已吸收 Open_Questions.md 全部回答（Q1–Q10）。

## 约定

- `[SU]` = 仅 Super User 可见
- `[U]` = 仅普通用户可见
- `[Both]` = 两类角色均可见
- `→` = 跳转到

## 业务规则速查

| 规则 | 说明 |
|------|------|
| 合同「即将出保」触发阈值 | 距到期 4 个月，触发三次：-4 个月、-1 个月、到期当天 |
| 合同状态文案 | 正常 → **在保**；即将到期 → **即将出保**；已过期 → **已出保**（禁止使用「脱保」「保障中」） |
| 设备「待验收」状态 | 新增状态。装机完成后未被验收的设备进入待验收状态；验收前设备属"无保"；每3个月发送验收提醒 |
| 合同展示日期 | 展示系统原始日期（装机 6 个月后才向客户可见，前台加注提示） |
| 合同类型文案 | Warranty → 保修服务；CSA → 白金保 / 白金保 Plus；POS → 按次付费服务；无合同 → 暂无服务合同 |
| 合同无「服务等级」概念 | 一台设备按时间线顺序经历多份合同（Warranty→POS→CSA），以倒序时间线展示 |
| PM 风险提醒触发 | 距上一次保养超过阈值天数且无新保养记录；**若设备在保且有下次保养计划则不触发** |
| PM 单次完成通知 | 不触发消息通知 |
| PM「正常」状态 | 不需要强调展示；只在异常（高风险）时显示风险提示 |
| 消息推送合并策略 | 多设备提醒合并发送：合同到期/PM计划/PM风险 → 单条消息列表展示多台设备，降低推送打扰率 |
| 设备卡片显示规则 | 只显示异常状态 tag；正常设备不显示「正常运行」tag；不显示设备类型文字（如磁共振成像系统） |
| 设备自定义备注 | 用户可在设备详情页为设备填写自定义名称；同步至设备列表卡片（以备注名为主，原名弱化）；不写回 D365 |
| 设备科室/位置自填 | 用户可在设备详情页自行填写/修改科室和设备位置；不写回 D365 系统 |
| 设备卡片状态优先级 | 已出保 > 即将出保 > 待验收 > 报修中 > PM高风险 > 本月有PM > 正常（正常不显示）|
| Super User 工单 | 数据范围更广，状态 Tab 与普通用户相同；SU 可见他人工单但不能替他人签字 |
| 工单与报修记录的区别 | 工单（工单对象）≠ 报修记录；一条报修可对应多个工单；保养记录对应保养工单；结构一致 |
| PDF / 邮箱动作 | Phase I 功能，原型应实现 |

## P1｜报修页（Tab 1）

**对应 Tab**：报修  
**两类角色均使用此页，但数据范围不同。**

### 固定顶部区域
- 页面标题「报修」
- 扫码报修入口行 `[Both]`
  - 图标 + 「扫码报修」+ 「输入设备编号 →」
  - 功能：进入 → P1a 扫码/输入报修页

### 筛选区域
- 搜索框：设备名称 / 医院名称
- 状态筛选 chips（单选）：全部 / 服务中 / 已完成 / 已取消
- 时间范围筛选：近三个月 / 近六个月 / 近一年（可选择一项）
- `[SU]` 额外筛选：医院 / 院区（以下拉或 sheet 实现）

### 列表区域
- 按月份分组，倒序排列
- 月份标题：「2026年4月（N条）」
- 每条报修卡片包含：
  - 设备名称（粗体）
  - 医院名称 + 科室/院区
  - 当前状态标签（服务中 / 服务完成 / 已取消 / 等）
  - 最新进展节点摘要（一行文字）
  - 工程师信息（如已分配：姓名 + 角色）
  - `[SU]` 可选：全渠道标识（若非本人报修的记录）
  - 底部动作行：
    - `[设备详情]` → P5 设备详情页
    - `[报修详情]` → P1b 报修详情页

### 空状态
- 无报修记录时：显示引导文案 + 扫码报修按钮

### 设计约束
- Super User 报修页**不做重 Dashboard**，不展示全院摘要数字卡片
- 报修页不展示合同信息，无论哪种角色

## P1a｜扫码/输入报修页

**入口**：P1 报修页顶部固定入口  
**功能**：提交新报修

- 扫码识别设备（调起微信相机）
- 或手动输入设备编号
- 识别成功后带出设备信息（名称、医院、EQ、序列号）
- 报修表单：
  - 是否造成伤害（必填）
  - 是否停机（必填）
  - 故障描述
  - 附件上传（照片/视频/录音）
  - 联系人信息
- 提交报修

## P1b｜报修详情页

**入口**：P1 报修卡片「报修详情」  
**功能**：查看单条报修的进展时间线

- 报修编号
- 报修设备信息（可跳转 → P5 设备详情）
- 服务工程师（姓名、角色、联系方式）
- 进展时间线（节点列表，倒序）：
  - 节点名称（报修已收到 / 已分配工程师 / 备件已出库 / 服务完成 等）
  - 时间戳
  - 节点说明文字

## P3｜设备页（Tab 2）——Super User

**对应 Tab**：设备  
**仅 Super User 视图**

### 快速筛选 chips（PM schedule 强入口）

| Chip 标签 | 筛选逻辑 |
|-----------|---------|
| 全部设备 | 无过滤 |
| **本月保养** | D365 推送的 PM schedule 中，计划日期在当月内（日历月）；必须有具体日期才纳入 |
| 即将出保 | 合同结束日期在 4 个月内（120 天）|
| 风险设备 | 有 PM 风险标记 或 已出保 |
| 报修中 | 当前有进行中报修 |

说明：「本月保养」chip 是 PM schedule 的首要感知入口，不可缺失。

### 搜索区域
- 搜索框：设备名称 / 科室 / 院区

### 设备列表
- 设备卡片（按优先级展示状态标签，可显示多个）：
  - 设备名称（粗体）；若有自定义备注，以备注名为主，原设备名以小字弱化
  - **不显示设备类型文字**（如「磁共振成像系统」「血管造影系统」）
  - 科室 + 院区
  - **状态标签（优先级：已出保 > 无保 > 即将出保 > 待验收 > 报修中 > PM高风险 > 本月有PM > 正常不显示）**：
    - 已出保：红色 tag，文案「已出保」
    - 无保：系统内设备合同类型为unknown的，黑色 tag，文案「无保」
    - 即将出保：橙色 tag，文案「即将出保」
    - 待验收：橙色 tag，文案「待验收」
    - 报修中：蓝色 tag，文案「报修中」
    - PM风险：橙色 tag，文案「PM 风险」
    - 本月有PM：蓝色 tag，文案「MM月DD日保养」（含具体日期）
    - **正常设备：不显示任何状态 tag**

- 点击 → P5' 设备详情（Super User 版）

- 「绑定设备」按钮：扫码识别并绑定设备

## P5｜设备详情页——普通用户

**入口**：P2 设备卡片 / P1 报修卡片 [设备详情]

### 固定顶部区域（蓝色背景，可折叠）
- 返回按钮
- 设备名称（若有自定义备注，以备注名为主）
  - 设备名支持用户自定义：点击编辑 → 输入自定义备注 → 保存（仅本地，不写回 D365）
  - 有备注时，原设备名以小字展示
- 展开/折叠后展示：所属医院、科室/位置、EQ 号、序列号
  - 科室/位置支持用户自行填写修改（本地保存，不写回 D365）
- 异常状态 chip（只显示异常）：
  - 报修中（有进行中服务）
  - PM高风险（无保且保养间隔超阈值）
- 快捷操作按钮（始终在顶部可见）极速报修为Primary, 弱化「电话咨询」，要尽可能引导用户用小程序报修，而不是打电话：
  - 「电话咨询」→ tel: 飞利浦服务热线
  - 「极速报修」→ 进入报修流程，填写报修问卷

### 内容区域（Tab 展示，Tab 顺序如下）

**Tab 1：全部工单** `[Both]`
- 包含该设备下所有工单：维修、保养、安装、FCO 4类
- 按工单类型分组（维修 / 保养 / 安装 / FCO）
- 每条工单：工单编号 + 类型 + 时间 + 状态，可点击查看工单详情（→ P4 工单详情页）
- **普通用户权限提示（顶部固定提示条）**：「仅显示与您账号相关的工单，完整记录请联系管理员查看」

**Tab 2：报修记录** `[Both]`
- 本人相关报修记录，按时间倒序
- 每条：报修时间 + 状态标签 + 进展摘要，可点击查看报修详情（→ P1b）
- **普通用户权限提示**：「仅显示与您账号相关的报修记录，完整记录请联系管理员查看」

**Tab 3：保养记录** `[Both]`
- 总结展示保养状态：上次保养日期、下次计划日期（若有）、距上次保养天数摘要，PM 风险提示
- 关联展示「保养工单」，以时间线倒序展示，可点击打开工单详情
- PM 风险提示（条件显示）：
  - 触发条件：设备无保 且 距上次保养超过阈值天数
  - 若设备在保且有下次保养计划，**不触发** PM 风险提示
  - 文案示例：「该设备已超过 X 个月未进行保养，建议关注」
- **普通用户权限提示**：「仅显示与您账号相关的保养工单，完整记录请联系管理员查看」

**不显示**：合同信息，任何合同相关字段（包括保障状态、到期日期）

## P5'｜设备详情页——Super User

**入口**：P3 设备卡片

### 固定顶部区域（蓝色背景，可折叠）
- 返回按钮
- 设备名称（若有自定义备注，以备注名为主）
  - 设备名支持用户自定义备注（本地保存，不写回 D365）
- 展开/折叠后展示：所属医院、科室/位置（用户可自填）、EQ 号、序列号
- 异常状态 chip（只显示异常）：
  - 已出保 / 即将出保 / 待验收
  - 报修中（有进行中服务）
  - PM高风险
- 快捷操作按钮，极速报修为Primary, 弱化「电话咨询」，要尽可能引导用户用小程序报修，而不是打电话：：
  - 「电话咨询」→ tel: 飞利浦服务热线
  - 「极速报修」→ 进入报修流程

### 内容区域（Tab 展示，Tab 顺序如下）

**Tab 1：全部工单** `[Both]`
- 包含该设备下所有工单：维修、保养、安装、FCO 4类
- 按工单类型分组
- 每条工单：工单编号 + 类型 + 时间 + 状态，可点击查看工单详情
- 不需要普通用户权限提示（SU 可见全部）

**Tab 2：报修记录** `[Both]`
- 全渠道报修记录（非仅本人），按时间倒序
- 每条：报修时间 + 状态标签 + 进展摘要，可点击查看报修详情

**Tab 3：保养工单** `[Both]`
- 以时间线展示历次保养记录
- 每条保养记录关联到保养工单，可点击打开工单详情
- 标题使用「保养工单」
- PM 风险提示（条件显示）：
  - 触发条件：距上次保养超过阈值天数（阈值=100天）且无新保养记录
  - **若设备在保且有下次保养计划，不触发 PM 风险提示**
  - **「PM正常」状态不显示** — 正常时直接省略提示

**Tab 4：合同信息** `[SU only]`
- **无「服务等级」概念**
- 合同以**时间线倒序**展示（最近的合同在最上方）：
  - 一台设备可经历多份合同（如 Warranty → POS → CSA）
  - 每份合同：类型文案（保修服务/按次付费服务/白金保）+ 起止日期 + 状态
- 当前合同状态 chip：在保（绿）/ 即将出保（橙）/ 已出保（红）
  - 注：展示系统原始日期；装机未满 6 个月时前台加注「合同信息待确认」

## P6｜工单页（Tab 3）

**两类角色均使用此页，状态 Tab 结构相同，数据范围不同。**

**角色差异说明**：Super User 可见他人的工单（数据范围更广），但工单签字动作仍由被要求签字的当事人执行；SU 只能查看，不能替他人签字。

### 搜索区域
- 搜索框：设备名称 / 工单编号
- `[SU]` 额外：医院 / 院区

### 筛选区域
- 状态 tabs（横向滑动，两类角色相同）：全部 / 待签字 / 已完成 / 已失效
- 类型筛选（可选下拉）：全部类型 / 维修 / 保养 / 安装 / FCO
- 时间范围：近三个月 / 近六个月 / 近一年
- `[SU]` 额外筛选：医院 / 院区（以下拉或 sheet 实现）

### 工单列表
- 按月份分组
- 工单卡片：
  - 工单类型标签（信号色：维修=error / 保养=success / FCO=caution / 安装=warning）
  - 设备名称
  - 医院名称 `[SU 必须显示以区分院区]`
  - 工单编号 + 请求时间
  - **动作区（按状态和角色）**：
    - 待签字（本人需签）→ `[去签字]` 主要按钮
    - 待签字（SU 查看他人工单，非本人签字）→ `[查看详情]` 次要入口，不显示「去签字」
    - 已完成 → `[评价服务]` `[下载 PDF]` `[发送邮箱]`（Phase I 功能，应实现）
    - 已失效 → 状态说明文字，无操作按钮

### 空状态
- 无工单时显示引导文案

## P7｜我的页（Tab 4）

**两类角色共用此页，内容按角色差异显示。**

### Hero 区
- 头像（姓名首字）
- 姓名
- 角色标签（Super User · [职位] / 普通用户）
  - 审核通过后自动更新

### 消息中心入口（Hero 区下方，优先位置）
- 行式入口：「消息中心」+ 未读数角标（有未读消息时显示数字）
- 点击 → P8 消息中心页

### 账户信息区块
- 所属医院
- 科室
- 手机号（脱敏）

### 权限信息区块
- 角色：Super User / 普通用户
- 权限范围：
  - `[SU]`：显示授权院区范围（如「WeConnect 医院 · 全院」）
  - `[U]`：「与本人相关的设备及服务记录」
- `[SU]` 权限有效期
- **不显示「合同信息可见：否」**：不向普通用户强调无法看到的权限

### `[U]` 申请升级为 Super User（独立功能区块）

此区块不放在「权限信息」内，作为独立、视觉明显的功能块展示。

**区块说明文案**：Super User 可查看授权范围内全院设备、合同保障信息和 PM 风险提醒。申请后由医院管理员审核。

**状态一：未提交**
- 标题：「申请升级账号权限」
- 说明：Super User 可见内容差异简述
- `[申请升级账号]` 主要按钮

**状态二：已提交，审核中**
- 「申请已提交」
- 申请日期：「提交于 YYYY-MM-DD」
- 状态文字：「等待管理员审核，审核结果将通过消息中心通知」

**状态三：审核通过**（由系统自动处理，不在此状态停留）
- 消息中心收到通知
- Hero 区角色标签自动更新为 Super User
- 数据权限自动生效

### 常用设备管理
- 行式入口：「常用设备管理」→ 编辑常用设备列表

### 设置区块
- 消息通知设置
- 帮助与反馈
- 退出登录

## P8｜消息中心页

**入口**：P7「我的」页中消息中心入口行  
**不作为底部 Tab。**

### 顶部操作区
- 页面标题「消息中心（N条未读）」
- `[全部已读]` 按钮（有未读消息时出现）
- `[批量删除]` 按钮（有消息时出现）

### 筛选区域
- 已读状态 chips（单选）：全部 / 未读 / 已读
- 消息类型 chips（可多选，或以下拉实现）：
  - 全部
  - 合同提醒（`[SU]`）
  - 验收提醒（`[SU]`）
  - PM 计划（`[SU]`）
  - PM 风险
  - 服务提醒

### 消息列表
- 时间分组：今日 / 本周 / 更早

**普通消息（单条）**：
- 图标圆圈（颜色按消息类型区分）
- 消息类别 badge（合同提醒 / 验收提醒 / PM 风险 / 服务提醒 等）
- 消息标题（未读时加粗 + 红点）
- 消息摘要（2 行截断）
- 时间
- 单条删除按钮
- 点击 → 标记已读 + 跳转到相关设备详情或工单详情

**聚合消息（多台设备合并为一条）**：
- 消息类别 badge + 设备数量角标（如「PM 风险 · 4台」）
- 消息标题（如「您有 4 台设备 PM 风险需关注」）
- 设备列表（默认折叠，显示前 2 台 + 「展开」按钮）
  - 展开后：列出每台设备名 + 所属科室 + 简短风险摘要
  - 每台设备可单独点击 → 跳转至该设备详情页
- 时间
- 单条删除按钮（删除整条聚合消息）

### 消息类型

| 类型 | 角色可见 | 触发条件 | 推送策略 | 优先级 |
|------|---------|---------|---------|-------|
| 合同出保提醒 | SU only | 合同已到期 | 实时，同日期多设备合并为一条 | 高 |
| 合同即将出保 | SU only | 距到期 4 个月 / 1 个月 / 到期当天（共 3 次）| 定时推送，同日期多设备合并为一条 | 高 |
| 设备待验收提醒 | SU only | 设备在待验收状态 > 90 天 | 每 3 个月推送，同医院多设备合并为一条 | 中 |
| PM 月度计划提醒 | SU only | 每月初查询 D365 PM 计划 | 月度批次，该月所有 PM 按医院/区域合并为一条 | 中 |
| PM 风险提醒 | **Both** | 距上次保养超过阈值且无新记录（用户有权限的设备）| 每月初批次检查，多设备按医院/区域合并为一条 | 中 |
| 权限升级审核通过 | `[U]` only | 管理员审批通过 | 实时，单条推送 | 高 |
//| 服务关键节点提醒 | Both | 工单重要状态变更（已派工 / 待签字 / 已完成）| 实时，单条推送 | 低 |

**注意**：
- 单次 PM 完成不触发通知
- PM 风险提醒面向两类角色，但普通用户仅见自己有权限的设备
- 多设备提醒合并策略：同日期同类型的提醒 → 单条消息中列表展示所有相关设备，用户可点击单台设备查看详情

### 交互规则
- 单条消息点击 → 标记已读 + 跳转到相关设备详情或工单详情
- 聚合消息点击 → 展开/收起设备列表；不跳转到单一页面
- 聚合消息内单台设备点击 → 标记该消息已读 + 跳转至该设备详情
- 单条删除：点击「删除」按钮删除该条消息
- 批量删除：进入选中模式，可勾选多条（含聚合消息）后删除

### 空状态
- 当前筛选条件下无消息：显示「暂无消息」+ 筛选器清除提示
- 全量无消息：显示「暂无消息」
