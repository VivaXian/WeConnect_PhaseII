# Page_Spec.md
# WeConnect Phase II｜页面规格说明

> 本文件描述每个页面的内容结构、交互原则和角色差异。不涉及视觉设计细节，不规定组件 API。  
> 最后更新：已吸收 Open_Questions.md 全部回答（Q1–Q10）。

---

## 约定

- `[SU]` = 仅 Super User 可见
- `[U]` = 仅普通用户可见
- `[Both]` = 两类角色均可见
- `→` = 跳转到

---

## 业务规则速查

| 规则 | 说明 |
|------|------|
| 合同「即将脱保」触发阈值 | 距到期 4 个月，触发三次：-4 个月、-1 个月、到期当天 |
| 合同展示日期 | 展示系统原始日期（装机 6 个月后才向客户可见，前台加注提示） |
| 合同类型文案 | Warranty → 保修服务；CSA → 白金保 / 白金保 Plus；POS → 按次付费服务；无合同 → 暂无服务合同 |
| PM 风险提醒触发 | 距上一次保养超过阈值天数且无新保养记录，两类角色均可收到（仅针对自己有权限的设备） |
| PM 单次完成通知 | 不触发消息通知 |
| 设备卡片状态优先级 | 脱保 > 即将脱保 > 报修中 > 有PM风险 > 本月有PM > 正常 |
| Super User 工单 | 数据范围更广，状态 Tab 与普通用户相同；SU 可见他人工单但不能替他人签字 |
| PDF / 邮箱动作 | Phase I 功能，原型应实现 |

## P1｜报修页（Tab 1）

**对应 Tab**：报修  
**两类角色均使用此页，但数据范围不同。**

### 固定顶部区域
- 页面标题「报修」
- 扫码报修入口行 `[Both]`
  - 图标 + 「扫码报修」+ 「输入设备编号 →」
  - 功能：进入 → P1a 扫码/输入报修页

### 筛选区域
- 搜索框：设备名称 / 医院名称
- 状态筛选 chips（单选）：全部 / 服务中 / 待签字（已完成待签）/ 已完成 / 已取消
- 时间范围筛选：近三个月 / 近六个月 / 近一年（可选择一项）
- `[SU]` 额外筛选：医院 / 院区（以下拉或 sheet 实现）

### 列表区域
- 按月份分组，倒序排列
- 月份标题：「2026年4月（N条）」
- 每条报修卡片包含：
  - 设备名称（粗体）
  - 医院名称 + 科室/院区
  - 当前状态标签（服务中 / 服务完成 / 已取消 / 等）
  - 最新进展节点摘要（一行文字）
  - 工程师信息（如已分配：姓名 + 角色）
  - `[SU]` 可选：全渠道标识（若非本人报修的记录）
  - 底部动作行：
    - `[设备详情]` → P5 设备详情页
    - `[报修详情]` → P1b 报修详情页

### 空状态
- 无报修记录时：显示引导文案 + 扫码报修按钮

### 设计约束
- Super User 报修页**不做重 Dashboard**，不展示全院摘要数字卡片
- 报修页不展示合同信息，无论哪种角色

## P1a｜扫码/输入报修页

**入口**：P1 报修页顶部固定入口  
**功能**：提交新报修

- 扫码识别设备（调起微信相机）
- 或手动输入设备编号
- 识别成功后带出设备信息（名称、医院、EQ、序列号）
- 报修表单：
  - 是否造成伤害（必填）
  - 是否停机（必填）
  - 故障描述
  - 附件上传（照片/视频/录音）
  - 联系人信息
- 提交报修

---

## P1b｜报修详情页

**入口**：P1 报修卡片「报修详情」  
**功能**：查看单条报修的进展时间线

- 报修编号
- 报修设备信息（可跳转 → P5 设备详情）
- 服务工程师（姓名、角色、联系方式）
- 进展时间线（节点列表，倒序）：
  - 节点名称（报修已收到 / 已分配工程师 / 备件已出库 / 服务完成 等）
  - 时间戳
  - 节点说明文字

## P2｜设备页（Tab 2）——普通用户

**对应 Tab**：设备  
**仅普通用户视图**

### 搜索区域
- 搜索框：设备名称

### 设备分组
- 我的设备（常用/最近使用/报修过的）
  - 设备卡片：设备名称 + 科室 + 当前服务状态
  - 「当前服务状态」优先显示：服务中 / 保养中 / 无进行中服务
  - 点击 → P5 设备详情（普通用户版）

- 「绑定设备」按钮：扫码识别并绑定设备

### 空状态
- 无绑定设备时：显示引导文案 + 扫码识别设备按钮

## P3｜设备页（Tab 2）——Super User

**对应 Tab**：设备  
**仅 Super User 视图**

### 快速筛选 chips（PM schedule 强入口）

| Chip 标签 | 筛选逻辑 |
|-----------|---------|
| 全部设备 | 无过滤 |
| **本月保养** | D365 推送的 PM schedule 中，计划日期在当月内（日历月）；必须有具体日期才纳入 |
| 即将脱保 | 合同结束日期在 4 个月内（120 天） |
| 风险设备 | 有 PM 风险标记 或 已脱保 |
| 报修中 | 当前有进行中报修 |

说明：「本月保养」chip 是 PM schedule 的首要感知入口，不可缺失。

### 搜索区域
- 搜索框：设备名称 / 科室 / 院区

### 设备列表
- 设备卡片（按优先级展示状态标签，可显示多个）：
  - 设备名称（粗体）
  - 科室 + 院区
  - **状态标签（优先级：脱保 > 即将脱保 > 报修中 > 有PM风险 > 本月有PM > 正常）**
    - 脱保：红色 tag，文案「暂无服务合同」
    - 即将脱保：橙色 tag，文案「合同将于 N 天后到期」
    - 报修中：蓝色 tag，文案「服务中」
    - 有PM风险：橙色 tag，文案「PM 风险」
    - 本月有PM：蓝色 tag，文案「本月有PM · MM-DD」（含具体日期）
    - 正常：不显示额外标签
  - 次要信息（小字）：合同截止日期（当处于即将脱保状态时显示）

- 点击 → P5' 设备详情（Super User 版）

- 「绑定设备」按钮：扫码识别并绑定设备

## P5｜设备详情页——普通用户

**入口**：P2 设备卡片 / P1 报修卡片 [设备详情]

### 固定顶部
- 返回按钮 + 设备名称 + 当前服务状态标签

### 内容区域（可滚动）

**区块 A：基础信息**
- 设备名称
- 所属医院
- 科室 / Location
- EQ 号
- 序列号

**区块 B：当前服务**（仅有进行中报修时显示）
- 当前报修状态 + 最新节点摘要
- → 查看报修详情

**区块 C：报修历史**
- 本人相关报修记录，按时间倒序
- 每条：报修时间 + 状态 + 摘要

**区块 D：保养信息** `[Both]`（由 D365 推送 PM 数据后显示）
- 上次保养日期（可关联 PM 工单入口，点击查看该次保养工单）
- 下次计划保养日期（如有 D365 schedule）
- 距上次保养天数摘要（「距上次保养已 X 天」）
- PM 风险提示（条件显示）：
  - 触发条件：距上次保养超过阈值天数且无新保养记录
  - 文案示例：「该设备已超过 X 个月未进行保养，建议关注」
  - 不向普通用户展示「脱保」相关合同信息，风险仅从保养间隔角度表达

**不显示**：合同信息，任何合同相关字段（包括保障状态、到期日期）

## P5'｜设备详情页——Super User

**入口**：P3 设备卡片

### 固定顶部
- 返回按钮 + 设备名称 + 主状态标签（优先级最高的当前状态）

### 警告 banner 区域（条件显示，可叠加）

| Banner | 触发条件 | 样式 |
|--------|---------|------|
| 已脱保 | 合同已到期 | danger（红）|
| 合同即将到期 | 距到期 ≤ 4 个月（120 天）| warning（橙）|
| PM 风险 | 距上次保养超过阈值天数且无新记录 | warning（橙）|

说明：多个 banner 同时满足时均展示，顺序：脱保 > 即将到期 > PM 风险。

### 内容区域（可滚动）

**区块 A：基础信息**
- 设备名称、所属医院、科室、EQ、序列号

**区块 B：合同信息** `[SU only]`
- 合同类型（客户可读文案）：
  - Warranty → 保修服务
  - CSA（白金保）→ 白金保 / 白金保 Plus（使用客户已知名称）
  - POS → 按次付费服务
  - 无合同 → 暂无服务合同
- 合同开始日期 / 结束日期
  - 注：展示系统原始日期；若设备装机未满 6 个月，前台加注说明「合同信息待确认」
- 合同状态 chip：
  - 保障中（绿）
  - 即将到期（橙，距到期 ≤ 4 个月）
  - 已脱保（红，已过期）
- 剩余天数数字 chip（正常 / 警戒 / 过期 三色）

**区块 C：保养信息** `[Both]`
- 上次保养日期（可关联 PM 工单，点击查看）
- 下次计划保养日期（由 D365 schedule 推送；无计划时不显示）
- 距上次保养天数摘要
- PM 风险状态：
  - 正常（距上次保养在阈值内）
  - 需关注（超过阈值且有合同）
  - 高风险（超过阈值且已脱保）

**区块 D：当前服务**（有进行中报修时显示）
- 当前报修状态 + 工程师 + → 查看报修详情

**区块 E：历史报修记录**（全渠道，非仅本人）
- 按时间倒序，每条：报修时间 + 类型 + 状态 + 摘要

## P6｜工单页（Tab 3）

**两类角色均使用此页，状态 Tab 结构相同，数据范围不同。**

**角色差异说明**：Super User 可见他人的工单（数据范围更广），但工单签字动作仍由被要求签字的当事人执行；SU 只能查看，不能替他人签字。

### 搜索区域
- 搜索框：设备名称 / 工单编号
- `[SU]` 额外：医院 / 院区

### 筛选区域
- 状态 tabs（横向滑动，两类角色相同）：全部 / 待签字 / 已完成 / 已失效
- 类型筛选（可选下拉）：全部类型 / 维修 / 保养 / 安装 / FCO
- 时间范围：近三个月 / 近六个月 / 近一年
- `[SU]` 额外筛选：医院 / 院区（以下拉或 sheet 实现）

### 工单列表
- 按月份分组
- 工单卡片：
  - 工单类型标签（信号色：维修=error / 保养=success / FCO=caution / 安装=warning）
  - 设备名称
  - 医院名称 `[SU 必须显示以区分院区]`
  - 工单编号 + 请求时间
  - **动作区（按状态和角色）**：
    - 待签字（本人需签）→ `[去签字]` 主要按钮
    - 待签字（SU 查看他人工单，非本人签字）→ `[查看详情]` 次要入口，不显示「去签字」
    - 已完成 → `[评价服务]` `[下载 PDF]` `[发送邮箱]`（Phase I 功能，应实现）
    - 已失效 → 状态说明文字，无操作按钮

### 空状态
- 无工单时显示引导文案

## P7｜我的页（Tab 4）

**两类角色共用此页，内容按角色差异显示。**

### Hero 区
- 头像（姓名首字）
- 姓名
- 角色标签（Super User · [职位] / 普通用户）
  - 审核通过后自动更新

### 消息中心入口（Hero 区下方，优先位置）
- 行式入口：「消息中心」+ 未读数角标（有未读消息时显示数字）
- 点击 → P8 消息中心页

### 账户信息区块
- 所属医院
- 科室
- 手机号（脱敏）

### 权限信息区块
- 角色：Super User / 普通用户
- 权限范围：
  - `[SU]`：显示授权院区范围（如「WeConnect 医院 · 全院」）
  - `[U]`：「与本人相关的设备及服务记录」
- `[SU]` 权限有效期
- **不显示「合同信息可见：否」**：不向普通用户强调无法看到的权限

### `[U]` 申请升级为 Super User（独立功能区块）

此区块不放在「权限信息」内，作为独立、视觉明显的功能块展示。

**区块说明文案**：Super User 可查看授权范围内全院设备、合同保障信息和 PM 风险提醒。申请后由医院管理员审核。

**状态一：未提交**
- 标题：「申请升级账号权限」
- 说明：Super User 可见内容差异简述
- `[申请升级账号]` 主要按钮

**状态二：已提交，审核中**
- 「申请已提交」
- 申请日期：「提交于 YYYY-MM-DD」
- 状态文字：「等待管理员审核，审核结果将通过消息中心通知」

**状态三：审核通过**（由系统自动处理，不在此状态停留）
- 消息中心收到通知
- Hero 区角色标签自动更新为 Super User
- 数据权限自动生效

### 常用设备管理
- 行式入口：「常用设备管理」→ 编辑常用设备列表

### 设置区块
- 消息通知设置
- 帮助与反馈
- 退出登录

## P8｜消息中心页

**入口**：P7「我的」页中消息中心入口行  
**不作为底部 Tab。**

### 顶部操作区
- 页面标题「消息中心（N条未读）」
- `[全部已读]` 按钮（有未读时出现）
- `[批量删除]` 按钮（有消息时出现）

### 消息列表
- 时间分组：今日 / 本周 / 更早
- 每条消息：
  - 图标圆圈（颜色区分类型）
  - 消息类别 badge
  - 消息标题（未读时加粗 + 未读圆点）
  - 消息摘要（2行截断）
  - 时间
  - 单条删除按钮

### 消息类型

| 类型 | 角色可见 | 触发条件 | 优先级 |
|------|---------|---------|-------|
| 设备脱保提醒 | SU only | 合同已到期 | 高 |
| 合同即将到期 | SU only | 距到期 4 个月 / 1 个月 / 到期当天（三次）| 高 |
| PM 风险提醒 | **Both** | 距上次保养超过阈值天数且无新记录（针对用户有权限的设备）| 中 |
| PM 月度计划提醒 | SU only | 每月 D365 推送月度 PM 计划 | 中 |
| 权限升级审核通过 | `[U]` only | 管理员审批通过后 | 高 |
| 服务关键节点提醒 | Both | 工单重要状态变更（已派工 / 待签字 / 已完成）| 低 |

**注意**：单次 PM 完成不触发通知；PM 风险提醒面向两类角色（仅自己有权限的设备）。

### 交互规则
- 点击消息 → 标记已读 + 跳转到相关设备详情或工单详情
- 单条删除：点击「删除」按钮删除该条消息
- 批量删除：选中多条（或全选）后删除

### 空状态
- 无消息时显示「暂无消息」

## 导航层级总结

```
底部导航
├── 报修（Tab 1）
│   ├── 报修列表（P1）
│   ├── 扫码/输入报修（P1a）
│   └── 报修详情（P1b）
├── 设备（Tab 2）
│   ├── 设备列表 - 普通用户（P2）或 Super User（P3）
│   └── 设备详情 - 普通用户（P5）或 Super User（P5'）
├── 工单（Tab 3）
│   └── 工单列表（P6）
└── 我的（Tab 4）
    ├── 我的页（P7）
    └── 消息中心（P8）
```
