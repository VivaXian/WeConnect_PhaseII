# Role_Comparison.md
# WeConnect Phase II｜角色对比

> 用途：快速对比两类角色在各 Tab 和各页面的可见内容与行为差异。

---

## 1. 角色定义回顾

| 维度 | 普通用户 | Super User |
|------|---------|------------|
| 信息范围 | 与本人相关（报修过或绑定过）的设备和服务 | 授权院区内及自己绑定过的所有设备和服务 |
| 使用模式 | 任务型（报修、签字、查进展）| 管理型（全院视图、风险感知、问题发现）|
| 合同信息 | 不可见 | 可见（仅设备维度）|
| PM 计划总览 | 不可见 | 可见（设备 Tab 强入口 + 消息中心）|
| PM 结果/风险 | 可见（本人相关设备）| 可见（全院范围）|

---

## 2. Tab 级别对比

### Tab 1：报修

| 内容项 | 普通用户 | Super User |
|--------|---------|------------|
| 扫码报修入口 | ✅ 显示，高优先级 | ✅ 显示，但优先级低于管理任务 |
| 报修记录数据范围 | 本人相关报修 | 授权范围内所有报修 |
| 搜索 | 设备名称 / 医院 | 设备名称 / 医院 / 院区 |
| 状态筛选 | 全部 / 服务中 / 待签字 / 已取消 | 全部 / 服务中 / 已完成 / 已取消 |
| 额外筛选 | 无 | 医院 / 院区筛选 |
| Dashboard 摘要 | 无 | 无（设计约束：不做重 Dashboard）|
| 合同信息 | 不显示 | 不显示（合同属于设备页）|
| 时间范围筛选 | ✅ 近三个月 / 近六个月 / 近一年 | ✅ 近三个月 / 近六个月 / 近一年 |

---

### Tab 2：设备

| 内容项 | 普通用户 | Super User |
|--------|---------|------------|
| 页面入口层 | 我的设备 / 最近 / 常用 | 全院设备列表 |
| 快速筛选 chips | 无（通过搜索过滤）| ✅「本月保养」「即将到期」「风险设备」「服务中」|
| 设备卡片信息 | 设备名 + 科室 + 当前服务状态 | 设备名 + 科室 + **单一主状态标签** |
| 设备卡片中的合同信息 | 不显示 | 可显示剩余天数（次要小字）|

---

### Tab 3：工单

| 内容项 | 普通用户 | Super User |
|--------|---------|------------|
| 工单数据范围 | 本人相关工单 | 授权范围内所有工单 |
| 状态 Tab | 全部 / 待签字 / 已完成 / 已失效 | 全部 / **待处理** / 待签字 / 已完成 / 已失效 |
| 类型筛选 | ✅（维修 / 保养 / 安装 / FCO）| ✅（相同，范围更广）|
| 医院 / 院区筛选 | 不需要 | ✅ |
| 工单动作 | 签字 / 评价 / PDF / 邮箱 | 同左（动作由工单状态决定，与角色无关）|
| 医院信息显示 | 通常只有一家医院，可省略 | 必须显示医院名称（区分院区）|

---

### Tab 4：我的

| 内容项 | 普通用户 | Super User |
|--------|---------|------------|
| 消息中心入口 | ✅（可见消息类型：PM结果、服务节点）| ✅（可见消息类型：合同 + PM计划 + PM风险 + 服务节点）|
| 角色标签 | 普通用户 | Super User · [职位] |
| 权限范围说明 | 与本人相关 | 显示授权院区、有效期 |
| 申请升级入口 | ✅ 显示 | 不显示（已是 SU）|
| 合同信息入口 | 不显示 | 不显示（合同在设备页，不在「我的」）|

## 3. 设备详情页对比

| 区块 | 普通用户 | Super User |
|------|---------|------------|
| 基础信息（名称/医院/科室/EQ/序列号）| ✅ | ✅ |
| 警告 Banner（脱保 / 到期 / PM风险）| 不显示 | ✅ |
| **合同信息区块** | ❌ 不显示 | ✅（类型 + 开始/结束日期 + 状态 + 剩余天数）|
| **PM 保养区块** | 显示结果/风险（本人相关设备）| ✅（上次保养 + 下次计划 + 风险状态）|
| 当前服务（进行中报修）| 本人相关报修 | 全渠道报修 |
| 历史报修记录 | 本人相关 | 全渠道 |
| 报修入口（从设备详情发起）| ✅ | ✅ |

## 5. 角色差异边界原则

以下是确定性规则，不应被实现时打破：

| 规则 | 说明 |
|------|------|
| 合同信息只给 Super User | 任何场景下普通用户都不可见合同字段 |
| 合同信息只在设备维度出现 | 不做独立合同 Tab，不在「我的」中放合同入口 |
| PM 月度计划只给 Super User | 普通用户只看 PM 结果通知和相关风险提醒 |
| 消息中心不作为底部 Tab | 两类角色的消息中心入口都在「我的」Tab 中 |
| Super User 报修页不做 Dashboard | 即使数据范围更广，展示形式仍为状态列表 |
| 设备卡片只突出单一主状态 | 不在卡片层堆叠合同+PM+报修状态，避免信息过载 |
| 工单 Tab 不改名为「待签工单」| 工单是对象+动作页，待签字是状态筛选项，不是页面定义 |

## 6. 角色切换（开发模拟用）

当前通过 `role-store.ts` 切换角色（`admin` = Super User，`user` = 普通用户）。

切换角色时，以下内容会随之变化：
- 底部导航视觉不变（两类角色相同结构）
- Tab 1 报修页：数据范围 + 额外筛选选项
- Tab 2 设备页：整体页面结构（普通用户视图 vs Super User 视图）
- 设备详情页：合同区块可见性 + 警告 banner 可见性 + 历史记录范围
- Tab 3 工单页：数据范围 + 状态 Tab 选项
- Tab 4 我的页：角色标签 + 权限说明 + 申请升级入口可见性
- 消息中心：可见消息类型范围
