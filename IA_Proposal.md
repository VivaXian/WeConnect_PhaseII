# IA_Proposal.md
# WeConnect Phase II｜信息架构提案

---

## 1. 当前 IA 评估

### 1.1 现状

当前代码实现的底部导航为：

```
工单 | 设备 | 消息 | 我的
```

普通用户的「工单」Tab 下存在二级 Tab：「工单 / 报修记录」。

### 1.2 问题清单

| # | 问题 | 影响 |
|---|------|------|
| 1 | 第一个 Tab 是「工单」而非「报修」 | 打断 Phase I 用户心智。Phase I 用户进入小程序首先找到的是报修进展，而不是工单签字。|
| 2 | 「消息」作为底部 Tab 独立存在 | 设计意图明确反对此做法。消息中心是弱提醒归档，不是核心任务入口，放底部导航会误导用户优先级认知。|
| 3 | 「报修记录」作为「工单」的二级 Tab | 逻辑颠倒。报修记录（修进展）是 Phase I 核心内容，应作为独立 Tab「报修」的主体，而非挂在「工单」下的子视图。|
| 4 | 「报修」入口消失 | 扫码报修和查看报修进展，是普通用户最高频的动作，不应被藏进二级 Tab。|
| 5 | PM schedule 无设备页强入口 | 月度 PM 计划目前只在消息中心可见，设计文件要求在「设备」Tab 有快速筛选入口。|
| 6 | Super User 的报修和工单页没有明确分离 | 目前 SuperUserServicePage 作为「工单」Tab 内容展示，实为报修记录，与工单签字动作混淆。|

### 1.3 评估结论

当前 IA **不合理**，需要重构 Tab 顺序和消息中心位置。这不是小调整，而是结构性修正。但现有页面组件（报修列表、工单列表、设备列表、消息中心、个人页）本身功能逻辑基本正确，只需重新编排到正确的 Tab 位置，不需要重写所有页面。

## 2. 推荐 IA

### 2.1 底部导航（两类角色相同结构）

```
报修 | 设备 | 工单 | 我的
```

#### Tab 设计原则
- **「报修」**：Phase I 用户心智延续，扫码报修 + 报修进展
- **「设备」**：Phase II 核心对象，普通用户看自己的设备，Super User 看全院视图
- **「工单」**：工单对象与收尾动作，不限于待签字
- **「我的」**：账户、权限、消息中心、设置——消息中心是内容，不是 Tab

### 2.2 各 Tab 职责

---
#### Tab 2：设备

**核心职责**：设备是 Phase II 复杂信息的核心容器，合同、PM、服务记录都绑定在设备下。

**普通用户**

```
设备页
├── 搜索：设备名称
├── 我的设备（常用 / 最近扫码 / 报修过的）
└── 设备卡片
    ├── 设备名称 + 科室
    └── 当前服务状态（如「服务中」「无进行中服务」）

设备详情（普通用户版）
├── 基础信息：名称、医院、科室、EQ、序列号
├── 当前服务（如有进行中报修）
├── 报修历史（本人相关）
└── PM 状态/风险提醒（如适用）

不显示：合同信息
```

**Super User**

```
设备页
├── 快速筛选（强入口）
│   ├── 全部设备
│   ├── 本月保养        ← PM schedule 强入口
│   ├── 即将到期
│   ├── 风险设备
│   └── 服务中
├── 搜索：设备名称 / 科室 / 院区
└── 设备卡片（展示全部状态标签）

设备详情（Super User 版）
├── 基础信息
├── 合同信息            ← 仅 Super User 可见
│   ├── 合同类型（使用客户可读文案）
│   ├── 合同有效期
│   └── 状态提示（保障中 / 即将到期 / 已脱保）
├── PM 状态             ← 设备页中重点呈现
│   ├── 上次保养日期
│   ├── 下次计划日期
│   └── 风险状态
├── 当前服务（进行中报修）
└── 历史报修记录（全渠道）
```

#### Tab 3：工单

**核心职责**：工单对象与收尾动作页，不只是待签字。

**普通用户**

```
工单页
├── 搜索：设备名称 / 工单编号
├── 状态 tabs：全部 / 待签字 / 已完成 / 已失效
└── 工单卡片（月份分组）
    ├── 工单类型标签：维修 / 保养 / 安装 / FCO
    ├── 设备名称 + 医院
    ├── 工单编号 + 请求时间
    └── 动作区：
        ├── 待签字 → [去签字] 按钮
        ├── 已完成 → [评价服务] [下载 PDF] [发送邮箱]
        └── 已失效 → 状态展示，无动作
```

**Super User**

```
工单页
├── 搜索：设备名称 / 工单编号 / 医院
├── 状态筛选：全部 / 待签字 / 待处理 / 已完成 / 已失效
├── 类型筛选：全部类型 / 维修 / 保养 / 安装 / FCO
├── 时间范围：近三个月 / 近六个月 / 近一年
└── 工单卡片（同普通用户结构，范围更广）
```

#### Tab 4：我的

**核心职责**：账户与工具收口，消息中心在此作为入口。

```
我的页
├── Hero 区：头像 / 姓名 / 角色标签
├── 消息中心入口
│   └── 显示未读数角标
├── 账户信息
│   ├── 所属医院
│   ├── 科室
│   └── 手机号
├── 权限信息
│   ├── 当前角色（普通用户 / Super User）
│   ├── 权限范围说明
│   └── Super User：授权范围、有效期
├── （普通用户）申请升级为 Super User 入口
├── 常用设备管理
└── 设置
    ├── 消息通知设置
    ├── 帮助与反馈
    └── 退出登录

→ 消息中心页（从「我的」进入）
    ├── 时间分组：今日 / 本周 / 更早
    ├── 消息类型分类
    ├── 已读 / 未读状态
    ├── 单条删除 + 批量删除
    └── 点击跳转：对应设备详情 / 工单详情
```

#### Tab 1：报修

**核心职责**：延续 Phase I 核心价值，让用户快速报修并查看报修进展。

**普通用户**

```
报修页
├── 顶部：扫码报修 入口（固定）
├── 搜索：设备名称 / 医院
├── 状态筛选 chips：全部 / 服务中 / 待签字 / 已取消
├── 时间筛选：近三个月 / 近六个月 / 近一年
└── 报修记录列表（月份分组）
    └── 报修卡片
        ├── 设备名称 + 医院/科室
        ├── 当前状态标签
        ├── 最新进展节点摘要
        ├── 工程师信息（如已分配）
        └── 快速跳转：[设备详情] [报修详情]
```

**Super User**

```
报修页
├── 顶部：报修入口（可保留，但优先级低于普通用户）
├── 搜索：设备名称 / 医院 / 院区
├── 状态筛选：全部 / 服务中 / 已完成 / 已取消
├── 时间筛选：近三个月 / 近六个月 / 近一年
└── 报修记录列表（月份分组）
    └── 报修卡片（内容与普通用户相同，范围更广）

注意：Super User 报修页仍以列表为主，不做重 Dashboard。
```

## 3. Tab 重构对比

| 位置 | 当前（错误）| 推荐（正确）| 原因 |
|------|------------|------------|------|
| Tab 1 | 工单 | **报修** | 延续 Phase I 第一入口 |
| Tab 2 | 设备 | 设备 | 正确，保留 |
| Tab 3 | 消息（错误）| **工单** | 工单是任务/动作页，该独立 Tab |
| Tab 4 | 我的 | 我的（包含消息中心入口）| 消息中心归入「我的」|

---

## 4. 消息中心位置变更说明

- **不放底部 Tab**：消息中心是弱提醒归档，放底部 Tab 误导用户优先级
- **不放顶部铃铛为主入口**：微信小程序顶部控件区空间有限，不可控
- **放在「我的」Tab 第一个区块**：位置显眼，带未读数角标，但不争抢底部导航位置
- **消息中心功能完整性不变**：分组、已读/未读、单条/批量删除、跳转功能保留

---

## 5. PM Schedule 强入口策略

PM schedule 不能只放在消息中心。推荐三个位置形成递进感知：

| 层级 | 位置 | 形式 |
|------|------|------|
| 发现 | 设备 Tab 快速筛选 chip | 「本月保养」chip，点击过滤本月有 PM 的设备 |
| 理解 | 设备详情 PM 区块 | 上次保养日期 + 下次计划日期 + 风险状态 |
| 提醒回看 | 消息中心 | PM 月计划提醒归档，点击跳转设备详情 |

## 6. 与当前代码的差异汇总

| 变更点 | 当前代码 | 推荐方向 |
|--------|---------|---------|
| Tab 1 | 工单（AppTab: orders）| 报修（AppTab: repair）|
| Tab 3 | 消息（AppTab: messages）| 工单（AppTab: orders）|
| Tab 4 | 我的（包含消息 Tab 角标）| 我的（包含消息中心入口行）|
| 普通用户工单 Tab 内容 | WorkOrderListPage（含报修记录二级 Tab）| 仅工单列表 |
| 普通用户报修 Tab 内容 | 缺失 | 报修记录 + 扫码入口 |
| Super User Tab 1 内容 | SuperUserServicePage（报修列表）| 移入正确的「报修」Tab |
| 消息中心 | 独立底部 Tab | 「我的」页中的入口行 + 消息中心子页 |
| PM schedule 设备入口 | 无 | 设备 Tab 快速筛选 chip「本月保养」|
